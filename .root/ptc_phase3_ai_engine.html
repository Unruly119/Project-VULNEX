AI Engine จะทำ 2 อย่าง:
1. สร้าง Prompt — แปลงข้อมูล JSON จาก Scanner เป็นคำถามที่ชัดเจนสำหรับ AI

STEP 02

เขียน prompt_builder.py

📄 src/prompt_builder.py

ไฟล์นี้คือ "ล่าม" ระหว่างข้อมูล JSON กับ AI — รับข้อมูลดิบจาก Scanner แปลงเป็นประโยคคำถามที่ Gemini เข้าใจได้ดีที่สุด

🎯
หลักการ Prompt ที่ดีในงาน Cybersecurity: ต้องบอก AI ว่า (1) บทบาทคืออะไร → (2) ข้อมูลที่มีคืออะไร → (3) ต้องการคำตอบแบบไหน ขาดส่วนใดส่วนหนึ่ง ผลลัพธ์จะกว้างเกินไปหรือไม่ตรงประเด็น
🧬 โครงสร้าง Prompt ที่เราจะสร้าง
ส่วนที่ 1 — Role (บทบาท)
คุณคือผู้เชี่ยวชาญด้าน Cybersecurity ที่ตรวจสอบความปลอดภัยเว็บไซต์สถานศึกษาไทย...
→ บอก AI ว่าตัวเองคือใคร ทำให้ตอบในมุมมองที่ถูกต้อง
ส่วนที่ 2 — Data (ข้อมูล)
ผลการสแกนเว็บไซต์ school.ac.th: - Security Score: 33/100 - Headers ที่ขาด: CSP, X-Frame-Options - SSL: ใช้ได้ เหลือ 45 วัน - External Scripts: 3 ตัว
→ ข้อมูลจริงจาก Scanner ที่แปลงให้อ่านง่าย
ส่วนที่ 3 — Output Format (รูปแบบคำตอบ)
ให้ตอบเป็นภาษาไทยเข้าใจง่าย แบ่งเป็น: 1. สรุปภาพรวม (2-3 ประโยค) 2. ปัญหาเร่งด่วน (bullet) 3. คำแนะนำแก้ไข (ทำได้เลยทันที)
→ กำหนดหน้าตาคำตอบ ทำให้แสดงผลใน UI ได้ง่าย

สร้างไฟล์ src/prompt_builder.py :

src/prompt_builder.py
1# src/prompt_builder.py — แปลง JSON → Prompt สำหรับ Gemini
2
3def build_prompt(scan_result: dict) -> str: function หลัก
4 """สร้าง prompt จากผล scan เพื่อส่งให้ Gemini"""
5
6 # ── ดึงข้อมูลจาก scan_result ──────────────────────
7 url = scan_result.get("url", "")
8 headers = scan_result.get("headers", {})
9 ssl = scan_result.get("ssl", {})
10 html = scan_result.get("html", {})
11
12 # ── ส่วนที่ 1: บทบาทของ AI ──────────────────────
13 role = """คุณคือผู้เชี่ยวชาญด้าน Cybersecurity สำหรับสถานศึกษาไทย \
14ที่ช่วยวิเคราะห์ความปลอดภัยเว็บไซต์และให้คำแนะนำภาษาไทยเข้าใจง่าย \
15สำหรับครูและบุคลากรฝ่ายไอทีที่ไม่มีความเชี่ยวชาญด้านความปลอดภัยเป็นพิเศษ"""
16
17 # ── ส่วนที่ 2: ข้อมูลจาก Scanner ────────────────
18 score = headers.get("score", 0)
19 missing = headers.get("headers_missing", [])
20 found = headers.get("headers_found", {})
21 ext_sc = html.get("external_scripts", [])
22 ins_fm = html.get("insecure_forms", [])
23 ssl_ok = ssl.get("valid", False)
24 days = ssl.get("days_left", 0)
25 ssl_warn= ssl.get("warning", "")
26
27 # แปลงรายการ list → ข้อความ bullet
28 missing_txt = ", ".join(missing) if missing else "ครบทุกตัว" join = เชื่อม list เป็น string
29 found_txt = ", ".join(found.keys()) if found else "ไม่มีเลย"
30 ext_txt = str(len(ext_sc)) + " ตัว"
31
32 # ── รวมเป็น Prompt เดียว ─────────────────────────
33 data_section = f"""
34ผลการตรวจสอบเว็บไซต์: {url}
35────────────────────────────────
36คะแนนความปลอดภัย: {score}/100
37Security Headers ที่มี: {found_txt}
38Security Headers ที่ขาด: {missing_txt}
39SSL Certificate: {"ปลอดภัย เหลือ " + str(days) + " วัน" if ssl_ok else "มีปัญหา!"}
40คำเตือน SSL: {ssl_warn if ssl_warn else "ไม่มี"}
41External Scripts: {ext_txt}
42Insecure Forms: {len(ins_fm)} ตัว
43"""
44
45 # ── ส่วนที่ 3: กำหนดรูปแบบคำตอบ ─────────────────
46 output_format = """
47กรุณาวิเคราะห์และตอบกลับเป็นภาษาไทย แบ่งเป็น 4 ส่วนชัดเจน:
48
49## 🔍 สรุปภาพรวม
50[อธิบายสถานะโดยรวม 2-3 ประโยค เหมาะสำหรับผู้บริหาร]
51
52## 🚨 ปัญหาเร่งด่วน (ต้องแก้ทันที)
53[bullet list ปัญหาที่อันตรายที่สุด พร้อมอธิบายว่าอันตรายอย่างไร]
54
55## 🛠️ คำแนะนำการแก้ไข
56[ขั้นตอนที่ทำได้จริง เรียงจากง่ายไปยาก]
57
58## ✅ จุดที่ดีแล้ว
59[สิ่งที่เว็บไซต์ทำได้ถูกต้อง เพื่อกำลังใจ]
60"""
61
62 # รวม 3 ส่วนเป็น prompt เดียว
63 return f"{role}\n\n{data_section}\n{output_format}" ส่งคืน string เดียว
🧠
ทำไมต้องแยก prompt_builder.py ออกมา? เพราะ Prompt อาจต้องแก้บ่อย เช่น เพิ่มภาษา เปลี่ยนรูปแบบ หรือเพิ่มข้อมูลใหม่ การแยกไฟล์ทำให้แก้ได้ง่าย โดยไม่ต้องแตะ logic อื่น — นี่คือหลัก Separation of Concerns ใน Software Engineering
STEP 03

เขียน ai_engine.py — หัวใจของระบบ

📄 src/ai_engine.py

ไฟล์นี้คือ "สะพาน" ระหว่างโปรเจกต์กับ Gemini API — รับ scan_result เข้า สร้าง prompt แล้วส่งให้ AI วิเคราะห์ พร้อมระบบ Cache ป้องกัน API call ซ้ำ

src/ai_engine.py
1# src/ai_engine.py — เชื่อมต่อ Gemini API
2import os
3from dotenv import load_dotenv โหลด .env file
4import google.generativeai as genai Gemini SDK
5from cachetools import TTLCache Cache หมดอายุอัตโนมัติ
6from prompt_builder import build_prompt import จากไฟล์ที่เพิ่งสร้าง
7
8# ── โหลด API Key จาก .env ────────────────────
9load_dotenv() อ่านไฟล์ .env
10genai.configure(api_key=os.getenv("GEMINI_API_KEY"))
11
12# ── ตั้งค่า Model ─────────────────────────────
13MODEL_NAME = "gemini-2.0-flash" เปลี่ยน model ที่นี่ที่เดียว
14model = genai.GenerativeModel(
15 MODEL_NAME,
16 generation_config={
17 "temperature": 0.3, ต่ำ = คำตอบแม่นยำ ไม่สร้างสรรค์เกิน
18 "max_output_tokens": 1500, จำกัดความยาวคำตอบ
19 }
20)
21
22# ── Cache: ไม่เรียก AI ซ้ำถ้าสแกน URL เดิม ──
23# maxsize=50 เก็บได้ 50 URL, ttl=3600 หมดอายุใน 1 ชั่วโมง
24_cache = TTLCache(maxsize=50, ttl=3600) ประหยัด API quota
25
26def analyze(scan_result: dict) -> dict: function ที่ app.py จะเรียก
27 """
28 รับ scan_result จาก scanner.py
29 ส่งคืน dict ที่มี analysis และ score สรุป
30 """
31 url = scan_result.get("url", "")
32
33 # ตรวจ cache ก่อน — ถ้ามีผลแล้วไม่ต้องเรียก API ใหม่
34 if url in _cache: hit cache!
35 return _cache[url]
36
37 result = {
38 "analysis": "", # คำวิเคราะห์จาก AI
39 "risk_level": "", # HIGH / MEDIUM / LOW
40 "score": 0, # คะแนน headers 0-100
41 "error": None
42 }
43
44 try:
45 # 1. สร้าง prompt จากข้อมูล scan
46 prompt = build_prompt(scan_result) เรียก prompt_builder
47
48 # 2. ส่ง prompt ให้ Gemini
49 response = model.generate_content(prompt) เรียก API !
50 result["analysis"] = response.text
51
52 # 3. คำนวณ risk level จากคะแนน
53 score = scan_result.get("headers", {}).get("score", 0)
54 result["score"] = score
55 if score >= 70: result["risk_level"] = "LOW" ปลอดภัย
56 elif score >= 40: result["risk_level"] = "MEDIUM" พอใช้
57 else: result["risk_level"] = "HIGH" อันตราย
58
59 # 4. บันทึก cache เพื่อไม่ต้องเรียก API ซ้ำ
60 _cache[url] = result save to cache
61
62 except Exception as e:
63 result["error"] = str(e)
64 result["analysis"] = f"❌ เรียก AI ไม่สำเร็จ: {e}"
65
66 return result
⚙️
temperature คืออะไร? (บรรทัด 17) ค่า 0.0 = ตอบแม่นตายตัวทุกครั้ง  |  ค่า 1.0 = สร้างสรรค์ ตอบต่างกันทุกครั้ง
สำหรับงาน Cybersecurity ต้องการความแม่นยำ จึงใช้ 0.3 — ตอบสม่ำเสมอ แต่ยังมีความหลากหลายบ้าง
