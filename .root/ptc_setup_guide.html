ติดตั้ง Python

Python คือภาษาที่เราจะใช้เขียนทั้งโปรเจกต์ ต้องติดตั้งก่อนเป็นอันดับแรก

WIN
ดาวน์โหลดและติดตั้ง Python บน Windows ⏱ ~5 นาที ▶
⚠️
สำคัญมาก! ตอนติดตั้ง ต้องติ๊กถูกที่ ☑ Add Python to PATH ก่อนกด Install ไม่งั้น command ไม่ทำงาน

ขั้นตอน:

1. ไปที่ python.org/downloads กด "Download Python 3.12.x"
2. เปิดไฟล์ที่ดาวน์โหลด (python-3.12.x-amd64.exe)
3. ✅ ติ๊ก "Add python.exe to PATH" ที่หน้าแรกของ installer
4. กด "Install Now" รอจนเสร็จ
5. กด "Close" เมื่อติดตั้งเสร็จ
✅ ตรวจสอบว่าติดตั้งสำเร็จ

เปิด Command Prompt (กด Win+R พิมพ์ cmd แล้ว Enter) แล้วพิมพ์:

python --version

ผลลัพธ์ที่ควรเห็น → Python 3.12.x

pip --version

ผลลัพธ์ที่ควรเห็น → pip 24.x from ...

💡
ถ้าขึ้น "python is not recognized" แปลว่าลืมติ๊ก Add to PATH ต้องถอนการติดตั้งแล้วทำใหม่ หรือเพิ่ม PATH ด้วยตัวเอง (บอก Advisor ได้เลยครับ)
MAC
ติดตั้ง Python บน macOS ⏱ ~8 นาที ▶
ℹ️
แนะนำใช้ Homebrew เพราะจัดการ package ง่ายกว่า และเป็นมาตรฐานของนักพัฒนา Mac
🍎 Terminal
# ขั้นที่ 1: ติดตั้ง Homebrew (ถ้ายังไม่มี) $ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # ขั้นที่ 2: ติดตั้ง Python ผ่าน Homebrew $ brew install python@3.12 # ขั้นที่ 3: ตรวจสอบ $ python3 --version # Python 3.12.x
⚠️
บน macOS ใช้ python3 ไม่ใช่ python (python คือ version 2 ที่ติดมากับระบบ)
LNX
ติดตั้ง Python บน Ubuntu/Linux ⏱ ~3 นาที ▶
🐧 Terminal
# อัปเดต package list ก่อน $ sudo apt update # ติดตั้ง Python 3.12 และ pip $ sudo apt install python3.12 python3-pip python3.12-venv -y # ตรวจสอบ $ python3 --version # Python 3.12.x

ติดตั้ง VS Code

VS Code คือโปรแกรมเขียนโค้ดที่ดีที่สุดสำหรับ Python ฟรี ใช้ง่าย รองรับภาษาไทย

ALL
ดาวน์โหลดและตั้งค่า VS Code ⏱ ~5 นาที ▶
1. ไปที่ code.visualstudio.com กด Download
2. ติดตั้งตามปกติ (กด Next ตลอด)
3. เปิด VS Code แล้วกด Ctrl+Shift+X เปิด Extensions
4. ค้นหา "Python" โดย Microsoft → กด Install
5. ค้นหา "Thai Language Pack" → กด Install (ถ้าต้องการเมนูภาษาไทย)
🎨
Shortcut ที่ใช้บ่อยที่สุด Ctrl+` → เปิด Terminal ใน VS Code
Ctrl+S → Save
F5 → Run โปรแกรม
Ctrl+/ → Comment โค้ด
PHASE 1 · STEP 3 OF 8

สร้าง Project Folder

จัดโครงสร้างไฟล์ให้เป็นระเบียบตั้งแต่แรก ทำให้ทำงานร่วมกันและ deploy ง่ายขึ้น

ALL
สร้างโครงสร้าง Project ⏱ ~2 นาที ▶
🪟 Command Prompt
# ไปที่ Desktop C:\> cd %USERPROFILE%\Desktop # สร้าง folder โปรเจกต์ C:\> mkdir ptc-ai-web-shield C:\> cd ptc-ai-web-shield # สร้าง folder ย่อย C:\> mkdir src tests assets
🍎 Terminal
$ cd ~/Desktop $ mkdir ptc-ai-web-shield $ cd ptc-ai-web-shield $ mkdir src tests assets
🐧 Terminal
$ cd ~/Desktop $ mkdir ptc-ai-web-shield && cd ptc-ai-web-shield $ mkdir src tests assets

โครงสร้างที่ควรได้:

ptc-ai-web-shield/ ├── src/ # โค้ดหลักทั้งหมด ├── tests/ # ไฟล์ทดสอบ ├── assets/ # รูปภาพ, logo ├── app.py # ← จะสร้างทีหลัง (Streamlit main) ├── requirements.txt # ← รายการ library └── .env # ← API Key (ห้าม upload GitHub!)
🔒
ไฟล์ .env สำคัญมาก! ไฟล์นี้จะเก็บ API Key ของ Gemini ห้ามอัปโหลดขึ้น GitHub เด็ดขาด จะสร้าง .gitignore ป้องกันในขั้นต่อไป
PHASE 1 · STEP 4 OF 8

Virtual Environment

venv คือ "กล่อง" แยกสำหรับแต่ละโปรเจกต์ ทำให้ library ของแต่ละโปรเจกต์ไม่ชนกัน — เป็น best practice ที่นักพัฒนาทุกคนทำ

ALL
สร้างและเปิดใช้ Virtual Environment ⏱ ~2 นาที ▶
🪟 Command Prompt (ใน folder โปรเจกต์)
# สร้าง virtual environment ชื่อ venv C:\...\ptc-ai-web-shield> python -m venv venv # เปิดใช้งาน (ทุกครั้งที่เปิด terminal ใหม่) C:\...\ptc-ai-web-shield> venv\Scripts\activate # จะเห็น (venv) นำหน้า prompt แสดงว่าสำเร็จ (venv) C:\...\ptc-ai-web-shield> ← แสดงว่า venv เปิดอยู่
🍎 Terminal
$ python3 -m venv venv $ source venv/bin/activate (venv) $ ← แสดงว่า venv เปิดอยู่
🐧 Terminal
$ python3 -m venv venv $ source venv/bin/activate (venv) $ ← แสดงว่า venv เปิดอยู่
💡
จำกฎนี้ไว้! ทุกครั้งที่เปิด terminal ใหม่ ต้อง activate venv ก่อนเสมอ ถ้าไม่เห็น (venv) นำหน้า แปลว่ายังไม่ได้ activate

ติดตั้ง Libraries

ติดตั้ง library ทั้งหมดที่โปรเจกต์ต้องใช้ในคำสั่งเดียว

ALL
สร้าง requirements.txt และติดตั้ง ⏱ ~3 นาที ▶

ขั้นที่ 1 — สร้างไฟล์ requirements.txt ใน folder โปรเจกต์:

📄 requirements.txt
# requirements.txt # วางข้อความนี้ในไฟล์ requirements.txt streamlit>=1.35.0 httpx>=0.27.0 beautifulsoup4>=4.12.0 google-generativeai>=0.8.0 python-whois>=0.9.0 cachetools>=5.3.0 lxml>=5.0.0 cryptography>=42.0.0 python-dotenv>=1.0.0

ขั้นที่ 2 — ติดตั้งทั้งหมดด้วยคำสั่งเดียว:

⚡ Terminal (ต้องอยู่ใน venv ก่อน)
# ตรวจสอบว่า venv เปิดอยู่ (ต้องเห็น (venv) นำหน้า) (venv) $ pip install -r requirements.txt # รอสักครู่... จะเห็น Successfully installed streamlit-1.x httpx-0.x ...
✅ ตรวจสอบ
pip list

ควรเห็น streamlit, httpx, beautifulsoup4, google-generativeai ในรายการ

PHASE 2 · STEP 6 OF 8

Gemini API Key

ขอ API Key จาก Google เพื่อให้โปรแกรมเรียกใช้ AI ได้ ฟรี ไม่ต้องใช้บัตรเครดิต

ALL
ขอ API Key และตั้งค่า .env ⏱ ~5 นาที ▶
1. ไปที่ aistudio.google.com (Login ด้วย Google Account)
2. กด "Get API Key" → "Create API Key in new project"
3. Copy API Key ที่ได้ (ขึ้นต้นด้วย AIza...)
4. สร้างไฟล์ .env ใน folder โปรเจกต์

เนื้อหาในไฟล์ .env:

🔑 .env
# ไฟล์ .env — ห้ามแชร์หรือ upload GitHub! GEMINI_API_KEY=AIzaXXXXXXXXXXXXXXXXXXXX # ← ใส่ key ของคุณ

สร้างไฟล์ .gitignore เพื่อป้องกัน .env หลุดไป GitHub:

🛡️ .gitignore
.env venv/ __pycache__/ *.pyc .DS_Store
🚨
API Key หลุด GitHub = ถูกแบน! Google จะ detect และ revoke key ทันทีถ้าอยู่บน public repo นี่เป็นข้อผิดพลาดที่พบบ่อยมากของ นักพัฒนาหน้าใหม่ .gitignore จะป้องกันได้
PHASE 3 · STEP 7 OF 8

Hello World Test

ทดสอบว่า Python, Gemini API และ library ทั้งหมดทำงานได้ก่อนเขียนโค้ดจริง

ALL
ทดสอบ Gemini API ครั้งแรก ⏱ ~3 นาที ▶

สร้างไฟล์ test_gemini.py ใน folder โปรเจกต์:

🐍 test_gemini.py
# test_gemini.py — ทดสอบ Gemini API import os from dotenv import load_dotenv import google.generativeai as genai # โหลด API Key จากไฟล์ .env load_dotenv() api_key = os.getenv("GEMINI_API_KEY") # ตั้งค่า Gemini genai.configure(api_key=api_key) model = genai.GenerativeModel("gemini-2.0-flash") # ส่งข้อความทดสอบ response = model.generate_content( "พูดว่า 'PTC AI Web Shield พร้อมใช้งาน!' เป็นภาษาไทย" ) print("✅ ตอบกลับจาก AI:") print(response.text)

รันทดสอบ:

▶ Terminal
(venv) $ python test_gemini.py # ผลลัพธ์ที่ควรเห็น: ✅ ตอบกลับจาก AI: PTC AI Web Shield พร้อมใช้งาน!
🎉
ถ้าเห็นข้อความตอบกลับ = Environment พร้อม 100%! Python ทำงานได้ → venv ทำงานได้ → library ติดตั้งถูก → Gemini API ใช้ได้
🔧
Error ที่พบบ่อยและวิธีแก้:
ModuleNotFoundError → ลืม activate venv หรือ pip install ยังไม่สำเร็จ
API_KEY_INVALID → API Key ผิด ตรวจสอบใน .env อีกครั้ง

Streamlit Test

ทดสอบ UI Framework ที่จะใช้สร้าง Dashboard ของโปรเจกต์

ALL
รัน Streamlit ครั้งแรก ⏱ ~2 นาที ▶

สร้างไฟล์ app.py (ไฟล์หลักของโปรเจกต์):

🎨 app.py — Streamlit Hello World
# app.py — หน้าแรกของ PTC AI Web Shield import streamlit as st # ตั้งค่าหน้า st.set_page_config( page_title="PTC AI Web Shield", page_icon="🛡️", layout="wide" ) # Header st.title("🛡️ PTC AI Web Shield") st.subheader("ระบบตรวจสอบความปลอดภัยเว็บไซต์ด้วย AI") st.divider() # รับ URL จากผู้ใช้ url = st.text_input( "🌐 ใส่ URL ที่ต้องการตรวจสอบ:", placeholder="https://example.com" ) if st.button("🔍 ตรวจสอบ", type="primary"): if url: st.success(f"✅ รับ URL: {url} — พร้อมเชื่อม AI!") else: st.warning("⚠️ กรุณาใส่ URL ก่อน")

รัน Streamlit:

▶ Terminal
(venv) $ streamlit run app.py # จะเปิดเบราว์เซอร์อัตโนมัติที่ Local URL: http://localhost:8501 Network URL: http://192.168.x.x:8501
🚀
เบราว์เซอร์เปิดและเห็นหน้า UI = Setup สำเร็จ 100%! Environment พร้อมทั้งหมด ขั้นต่อไปคือเริ่มเขียน logic จริง
